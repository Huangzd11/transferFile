# 14 — V0.0.3 平台向网关推送文件

## 1. 目标

平台将文件写入网关**已知路径**；网关不负责“猜路径”，仅按简报中的 `FullPathFileName` 创建/覆盖写入。

| 需求 | 实现要点 |
|------|----------|
| 先简报后内容 | 简报确认 `Status=0` 后平台才发内容；`Status=1` 则不发内容 |
| 每段确认 | 网关每收一段推送内容即发**内容确认** |
| 超时/断连 | 任一方等待对方下一帧超过 `timeoutSec`（默认 180s）或 MQTT 断开 → 关闭写句柄、会话失败 |
| 断点续传 | **不支持**（与召唤上传的 `StartByte` 无关） |

## 2. MQTT Topic（默认可配置）

`gatewayId=gw001` 时：

| 方向 | 业务 | Topic |
|------|------|-------|
| 平台 → 网关 | 推送文件简报 | `transfer/sim/gw001/platform/push/brief` |
| 网关 → 平台 | 简报确认 | `transfer/sim/gw001/gateway/push/brief_confirm` |
| 平台 → 网关 | 推送文件内容 | `transfer/sim/gw001/platform/push/content` |
| 网关 → 平台 | 内容确认 | `transfer/sim/gw001/gateway/push/content_confirm` |

## 3. 报文（JSON `Data`，字段均为 string）

### 3.1 推送文件简报（平台 → 网关）

```json
{
  "Data": {
    "CmdId": "1001",
    "FullPathFileName": "/data/transfer/app.bin",
    "FileCrc": "0x12345678",
    "FileSize": "4096",
    "ModifyTime": "2026-06-01 12:00:00"
  }
}
```

### 3.2 简报确认（网关 → 平台）

```json
{
  "Data": {
    "CmdId": "1001",
    "Status": "0",
    "ErrorCode": "",
    "Note": ""
  }
}
```

`Status`：`0` 成功，`1` 失败。失败时 `ErrorCode` 必填，**不再接收**该 CmdId 的内容帧。

### 3.3 推送文件内容（平台 → 网关）

```json
{
  "Data": {
    "CmdId": "1001",
    "FileSegNo": "1",
    "Content": "<Base64>",
    "Continue": "1"
  }
}
```

`FileSegNo` 从 `1` 递增；`Continue`：`1` 还有后续，`0` 最后一段。

### 3.4 内容确认（网关 → 平台）

```json
{
  "Data": {
    "CmdId": "1001",
    "FileSegNo": "1",
    "Status": "0",
    "ErrorCode": "",
    "Note": ""
  }
}
```

## 4. 网关实现

| 模块 | 路径 |
|------|------|
| 编解码 | `src/protocol/json_push_codec.cpp` |
| 接收编排 | `src/app/push_receive_orchestrator.cpp` |
| 推送会话 | `src/session/memory_push_session_store.cpp` |
| 写文件 | `FileStore::openWriteCreate` / `writeAt` |

流程摘要：

1. 收简报 → 路径校验 → 创建空文件 → 简报确认成功 → `watchdog.arm`
2. 收内容段 → 校验 `FileSegNo` → 顺序写入 → 内容确认成功 → `watchdog.reset`
3. `Continue=0` → 校验总大小与 CRC32 → 关闭句柄 → 移除会话
4. 超时 → `abortSession` 关闭句柄

与**召唤上传**互斥：推送进行中拒绝新召唤；召唤进行中拒绝新推送（`BUSY`）。

## 5. 联调命令

**目标机**（OpenWrt）：

```bash
./transferFile -c config/transferFile.gateway.target.json
```

**开发机**（平台模拟）：

```bash
# 本地准备待推送文件，gateway-path 为目标机上的保存路径（须在 allowedPathRoots 内）
./platform_sim -c config/transferFile.platform.json \
  --push-file ./my.bin --gateway-path /tmp/my.bin
```

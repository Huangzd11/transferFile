# 18 — 业务时序流程（召唤 / 推送）

**文档版本**：V0.0.4（2026-06-03）

本文档给出**召唤上传**与**平台推送**的 MQTT 交互时序图（含 Topic 与报文阶段）。报文字段详见 [04-通信协议.md](04-通信协议.md)；网关状态机详见 [03-业务流程与状态机.md](03-业务流程与状态机.md)。

默认 `gatewayId=gw001` 时 Topic 前缀为 `transfer/sim/gw001/`。

---

## 1. 召唤文件时序（平台召唤 → 网关上传）

**业务方向**：平台下发召唤，网关从本地读文件，先回**简报**，再按段上传**内容**；V0.0.4 起**每段内容**须收到平台**内容确认**后才发下一段。支持断点续传（`StartByte`，见 [06-断点续传设计.md](06-断点续传设计.md)）。

| 步骤 | 方向 | Topic | 报文 |
|------|------|-------|------|
| ① | 平台 → 网关 | `platform/summon` | 召唤文件 |
| ② | 网关 → 平台 | `gateway/brief` | 文件简报 |
| ③ | 网关 → 平台 | `gateway/content` | 文件内容（分段） |
| ④ | 平台 → 网关 | `platform/content_confirm` | 文件内容确认（V0.0.4） |

### 1.1 正常流程（简报成功 + 逐段确认）

```mermaid
sequenceDiagram
  autonumber
  participant P as 平台
  participant B as MQTT_Broker
  participant G as 网关_transferFile
  participant F as 本地文件

  P->>B: ① 召唤 summon<br/>CmdId, FullPathFileName, StartByte
  B->>G: onSummon
  G->>G: 校验路径/BUSY/偏移
  G->>F: 打开并读取元数据
  G->>B: ② 简报 brief Status=0<br/>FileSize, FileCrc, ModifyTime
  B->>P: 简报成功

  loop 直至 Continue=0（末段）
    G->>F: readAt(偏移, chunkSize)
    G->>B: ③ 内容 content<br/>FileSegNo, Content(Base64), Continue
    B->>P: 内容段 N
    Note over G: WaitingContentConfirm
    P->>B: ④ 内容确认 content_confirm<br/>FileSegNo, Status=0
    B->>G: onContentConfirm
    G->>G: 超时计时 reset；准备下一段
  end

  G->>G: 会话 Completed，关闭句柄
```

### 1.2 简报失败（R3：不发内容）

```mermaid
sequenceDiagram
  participant P as 平台
  participant B as MQTT_Broker
  participant G as 网关

  P->>B: 召唤 summon
  B->>G: onSummon
  G->>G: 文件不存在/路径非法/BUSY 等
  G->>B: 简报 brief Status=1<br/>ErrorCode
  B->>P: 简报失败
  Note over G: 不发布 content；会话结束
```

### 1.3 内容确认失败或超时（C3 / R4）

```mermaid
sequenceDiagram
  participant P as 平台
  participant B as MQTT_Broker
  participant G as 网关

  G->>B: 内容 content FileSegNo=N
  B->>P: 内容段 N

  alt 平台回复确认失败
    P->>B: content_confirm Status=1
    B->>G: onContentConfirm
    G->>G: 中止传输，关句柄，清会话
  else 超时未收到确认
    Note over G: timeoutSec 内无有效交互
    G->>G: Aborted，停止后续段
  end
```

### 1.4 断点续传（R5）

中断后平台**再次发布召唤**，相同 `CmdId`（建议）、相同 `FullPathFileName`，`StartByte` 为下一段起始位置（1-based）。网关校验通过后从该偏移继续，**FileSegNo 从 1 重新计数**。

```mermaid
sequenceDiagram
  participant P as 平台
  participant B as MQTT_Broker
  participant G as 网关

  Note over P,G: 首次传输在段 M 后超时/失败
  P->>B: 召唤 summon StartByte=已传+1
  B->>G: onSummon（续传校验）
  G->>B: 简报 brief Status=0
  B->>P: 简报
  loop 从续传偏移至文件尾
    G->>B: content FileSegNo=1,2,...
    B->>P: 内容
    P->>B: content_confirm Status=0
    B->>G: onContentConfirm
  end
```

### 1.5 与推送互斥

召唤会话活跃（含 `WaitingContentConfirm`）时，新推送简报返回 `BUSY`；反之推送进行中拒绝新召唤。见 [14-V0.0.3-平台推送.md](14-V0.0.3-平台推送.md) §4。

## 2. 推送文件时序（平台推送 → 网关落盘）

**业务方向**：平台将文件写入网关**已知路径**；先**推送简报**，网关**简报确认**成功后再发**推送内容**；网关每收一段即发**内容确认**。不支持断点续传。

| 步骤 | 方向 | Topic | 报文 |
|------|------|-------|------|
| ① | 平台 → 网关 | `platform/push/brief` | 推送文件简报 |
| ② | 网关 → 平台 | `gateway/push/brief_confirm` | 简报确认 |
| ③ | 平台 → 网关 | `platform/push/content` | 推送文件内容（分段） |
| ④ | 网关 → 平台 | `gateway/push/content_confirm` | 推送内容确认 |

### 2.1 正常流程

```mermaid
sequenceDiagram
  autonumber
  participant P as 平台
  participant B as MQTT_Broker
  participant G as 网关_transferFile
  participant F as 本地文件

  P->>B: ① 推送简报 push/brief<br/>CmdId, FullPathFileName, FileSize, FileCrc
  B->>G: onPushBrief
  G->>G: 路径白名单校验
  G->>F: openWriteCreate（创建/截断）
  G->>B: ② 简报确认 brief_confirm Status=0
  B->>P: 可接收内容
  Note over G: watchdog.arm

  loop 直至 Continue=0
    P->>B: ③ 推送内容 push/content<br/>FileSegNo, Content, Continue
    B->>G: onPushContent
    G->>G: 校验 FileSegNo 连续
    G->>F: writeAt 顺序写入
    G->>B: ④ 内容确认 push/content_confirm Status=0
    B->>P: 段 N 已收
    G->>G: watchdog.reset
  end

  G->>G: 校验 CRC/总大小，关闭句柄，移除会话
```

### 2.2 简报确认失败（P2）

```mermaid
sequenceDiagram
  participant P as 平台
  participant B as MQTT_Broker
  participant G as 网关

  P->>B: 推送简报 push/brief
  B->>G: onPushBrief
  G->>G: 路径非法/BUSY/无法创建文件
  G->>B: brief_confirm Status=1<br/>ErrorCode
  B->>P: 简报确认失败
  Note over G,P: 平台不得再发该 CmdId 的 push/content
```

### 2.3 段序号错误或确认失败（P3）

```mermaid
sequenceDiagram
  participant P as 平台
  participant B as MQTT_Broker
  participant G as 网关

  P->>B: push/content FileSegNo 非期望序号
  B->>G: onPushContent
  G->>G: abortSession，关写句柄
  G->>B: content_confirm Status=1（可选）
  B->>P: 失败
  Note over P: 须新 CmdId 全量重推（无续传）
```

### 2.4 超时（P3 / R4 同类规则）

推送会话在 `timeoutSec`（默认 180s）内若无**有效交互**（收到推送内容段或简报确认后的下一帧交互），网关 `abortSession` 并关闭写句柄。平台需使用**新 CmdId** 重新从简报开始全量推送。

## 3. 两业务对比

| 项 | 召唤上传 | 平台推送 |
|----|----------|----------|
| 数据流向 | 网关 → 平台 | 平台 → 网关 |
| 谁先动 | 平台发召唤 | 平台发推送简报 |
| 第一段应答 | 网关简报 | 网关简报确认 |
| 内容段确认方 | **平台**确认网关上传段 | **网关**确认平台推送段 |
| 断点续传 | 支持 `StartByte` | 不支持 |
| 编排模块 | `TransferOrchestrator` | `PushReceiveOrchestrator` |

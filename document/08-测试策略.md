# 08 — 测试策略

## 1. 文档目的

定义项目的 TDD 流程、测试分层、需求追溯及当前已实现的测试文件（含 **V0.0.2** 召唤验收与 **V0.0.3** 推送验收用例）。

## 2. TDD 工作流

```text
1. 编写失败测试（Red）
2. 最小实现（Green）
3. 重构（Refactor）
4. 重复
```

## 3. 测试分层

| 层次 | 范围 | 运行环境 |
|------|------|----------|
| 单元测试 | Codec、FileStore、Orchestrator、Watchdog、Config、RuntimeLog | 主机 `ctest` |
| 集成测试 | 模拟 MQTT 总线端到端 | 主机 `test_mqtt_simulation.cpp` |
| 冒烟 | 真实 Broker + 目标机网关 | 手工，见 10 章 |

交叉编译 **不** 构建 `transfer_tests`（`BUILD_TESTING=OFF`）。

## 4. 已实现测试文件

| 文件 | 覆盖 |
|------|------|
| `tests/unit/test_base64.cpp` | Base64 |

| `tests/unit/test_file_store.cpp` | 路径白名单、读文件 |

| `tests/unit/test_orchestrator_v002.cpp` | **V0.0.2** R3/R4/R5/大文件验收 |
| `tests/unit/test_timeout.cpp` | R4 超时 |
| `tests/unit/test_mqtt_simulation.cpp` | 模拟总线集成 |
| `tests/unit/test_config_loader.cpp` | 配置加载 |
| `tests/unit/test_runtime_log.cpp` | 日志格式与落盘 |
| `tests/unit/test_push_codec.cpp` | 推送 JSON 编解码 |

| `tests/unit/test_push_e2e.cpp` | 模拟总线推送端到端 |

运行：

```bash
./scripts/run-acceptance-tests.sh
# 或
./scripts/build-native.sh && cd build && ctest --output-on-failure
```

V0.0.3 起共 **42** 项自动化测试（V0.0.2 召唤 31 项 + V0.0.3 推送 9 项 + `test_build_info` 等，以 `transfer_tests` 输出为准）。

测试框架：系统 GTest 或 `tests/minimal_test.hpp`。

## 8. 冒烟测试检查表（目标机）

- [ ] `transferFile` 启动，日志含 `MQTT 已连接`
- [ ] 开发机 `platform_sim --gateway-file` 发召唤
- [ ] 目标机日志：`收到召唤` → `简报成功` → `传输完成`
- [ ] 平台侧 `联调成功` 且文件内容一致
- [ ] 目标机 `log/YYYY-MM-DD.log` 有对应记录
- [ ] （V0.0.3）`platform_sim --push-file` 推送成功，目标机 `md5sum` 一致（见 [15-V0.0.3-验收说明.md](15-V0.0.3-验收说明.md) §3）

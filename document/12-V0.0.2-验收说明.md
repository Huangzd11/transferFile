## 3. 目标机手工验收（R5 续传）

### 3.1 全量上传

目标机：

```bash
dd if=/dev/urandom of=/tmp/v002_test.bin bs=10240 count=1
./transferFile -c transferFile.gateway.target.json
```

开发机：

```bash
./build/platform_sim -c config/transferFile.platform.json \
  --gateway-file /tmp/v002_test.bin
cmp /tmp/platform_received_file.bin <(scp 回来的 v002_test.bin)  # 或对比 md5
```

### 3.2 断点续传（模拟平台只收前半段）

1. 第一次全量召唤，平台侧**只保存前 4096 字节**（或记下已收字节数 N）。
2. 第二次召唤同一文件、同一 `CmdId`，`StartByte = N + 1`（1-based）：

```bash
./build/platform_sim -c config/transferFile.platform.json \
  --gateway-file /tmp/v002_test.bin --start-byte 4097
```

3. 将第二次收到的内容**追加**到第一次保存的文件，与目标机源文件 `cmp` 一致。

### 3.3 R3 / R4 现场 spot check

| 项 | 操作 | 预期 |
|----|------|------|
| R3 | 召唤 `/etc/passwd` 或不存在路径 | 简报失败，网关无内容段 |
| R4 | 发召唤后 180s 内不再发新召唤（可临时改小 `timeoutSec`） | 网关日志「传输超时中止」 |

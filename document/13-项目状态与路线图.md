## 3. 配置对照

| 文件 | 使用方 | `brokerHost` 要点 |
|------|--------|-------------------|
| `transferFile.platform.json` | 开发机 platform_sim | `127.0.0.1`（Broker 在本机） |
| `transferFile.gateway.target.json` | 目标机 transferFile | 开发机局域网 IP（如 `192.168.10.137`） |
| `transferFile.mqtt-debug.json` | 本机冒烟 | 均为 `127.0.0.1` |

`gatewayId` 两边必须一致（默认 `gw001`）。

## 5. 建议的下一步（按优先级）

### P0 — 与真实平台对接（上线前必做）

1. **定稿 MQTT Topic、ErrorCode、allowedPathRoots**（替换 `transfer/sim/...` 占位）
2. 用**生产 Broker** 替换开发机 mosquitto；确认 TLS/鉴权需求
3. 平台正式客户端发召唤，对比 `platform_sim` 的 JSON 字段与时序
4. 目标机 **R5 续传** 手工验收：`--start-byte` + `cmp` 全文件

### P1 — 生产化

1. **开机自启**（procd / systemd）与配置文件固定路径（如 `/etc/transferFile/`）
2. **日志**：`log/` 目录轮转或接入 syslog（见 [11-运行日志.md](11-运行日志.md)）
3. 现场 **mosquitto 监听 0.0.0.0** 与防火墙 1883 规则文档化
4. 发布检查清单：召唤 [12-V0.0.2-验收说明.md](12-V0.0.2-验收说明.md) §5；推送 [15-V0.0.3-验收说明.md](15-V0.0.3-验收说明.md) §5

### P2 — 后续增强（按业务需要）

| 能力 | 说明 |
|------|------|
| MQTT TLS / 用户名密码 | 配置项已有占位，需实现连接逻辑 |
| 异步内容发送 | 超长文件降低单次 `onSummon` 阻塞；利于 R4 传输中超时 |
| 平台 ErrorCode 对齐 | 更新 `error_codes.hpp` 与文档 04 |
| 配置项 `logDir` | 日志路径可配置，避免写死 `log/` |
| CI | 主机 `run-acceptance-tests.sh` 接入流水线 |

### 明确不做（除非业务变更）

- 平台侧实现（本仓库仅网关）
- 多会话并发传输（当前 BUSY 单会话）
- 网关主动续传（续传由平台重召唤）

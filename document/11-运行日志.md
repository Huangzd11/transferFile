# 11 — 运行日志（RuntimeLog）

## 1. 文档目的

说明 `transferFile` 与 `platform_sim` 的**运行期业务日志**：输出格式、落盘规则、API、典型消息及联调排查方法。面向开发、测试与目标机运维人员。

架构中的模块位置见 [02-系统架构.md](02-系统架构.md)；接口摘要见 [05-模块与接口设计.md](05-模块与接口设计.md) 第 7 节（本节为完整说明）。

## 2. 适用范围

| 程序 | 是否使用 RuntimeLog | 说明 |
|------|---------------------|------|
| `transferFile` | 是 | 网关：MQTT、召唤、传输全流程 |
| `platform_sim` | 是 | 平台模拟：发召唤、收简报/内容 |
| `transfer_tests` | 否（仅单测临时目录） | 单元测试 `test_runtime_log` 使用 `/tmp/...` |

**不使用 RuntimeLog 的输出**（仍打印到 stdout/stderr）：

- 启动横幅：版本号、配置文件路径、超时、分块、`allowedPathRoots`、MQTT Broker/Topic 等
- 命令行帮助、配置加载失败、`mosquitto_loop` 异常等少量 `std::cerr` 错误
- `platform_sim` 下收到的原始 JSON 全文（`std::cout` 调试输出）

## 3. 日志行格式

每条日志**一行**，控制台与文件内容**完全一致**。

### 3.1 语法

```text
[YYYY-MM-DD HH:MM:SS.mmm] [源文件路径:行号] [LEVEL] 消息正文
```

### 3.2 字段说明

| 字段 | 示例 | 说明 |
|------|------|------|
| 时间 | `[2026-06-01 16:49:52.744]` | 本地时区；毫秒 3 位；方括号包裹 |
| 位置 | `[src/mqtt/mosquitto_mqtt_adapter.cpp:91]` | 编译期 `__builtin_FILE()` / `__builtin_LINE()`；优先保留 `src/` 起的路径 |
| 级别 | `[INFO]` / `[WARN]` / `[ERROR]` | 见下表 |
| 消息 | 中文描述 | 不含 JSON 载荷全文、不含文件二进制内容 |

时间与文件位置之间 **1 个空格**（`] [`）。

### 3.3 级别与输出流

| 级别 | 控制台 | 日志文件 |
|------|--------|----------|
| INFO | `stdout` | 写入 |
| WARN | `stderr` | 写入 |
| ERROR | `stderr` | 写入 |

### 3.4 完整示例

```text
[2026-06-01 16:49:52.744] [src/mqtt/mosquitto_mqtt_adapter.cpp:91] [INFO] 正在连接 MQTT Broker 127.0.0.1:1883 ...
[2026-06-01 16:49:52.801] [src/mqtt/mosquitto_mqtt_adapter.cpp:25] [INFO] MQTT 已连接 Broker，已订阅: transfer/sim/gw001/platform/summon
[2026-06-01 16:49:53.120] [src/app/transfer_orchestrator.cpp:110] [INFO] <<< 收到召唤报文，长度 90 字节
[2026-06-01 16:49:53.125] [src/app/transfer_orchestrator.cpp:181] [INFO] >>> 已发布简报成功 CmdId=8001 FileSize=5 FileCrc=0x3BB935C6 ModifyTime=2026-06-01 16:19:32
[2026-06-01 16:49:53.130] [src/app/transfer_orchestrator.cpp:207] [INFO] 开始发送文件内容...
[2026-06-01 16:49:53.145] [src/app/transfer_orchestrator.cpp:99] [INFO] 传输完成 CmdId=8001 总大小=5 字节
```

## 4. 文件落盘

### 4.1 目录与文件名

| 项 | 约定 |
|----|------|
| 目录 | 进程**当前工作目录**下的 `log/`（默认，见 `init`） |
| 文件名 | `YYYY-MM-DD.log`（按自然日，跨日自动切换新文件） |
| 模式 | 追加（`append`），不覆盖历史 |
| 创建 | 首次 `init` 时递归创建目录（POSIX `mkdir`，兼容 OpenWrt 交叉编译） |

示例：2026 年 6 月 1 日运行产生 `log/2026-06-01.log`。

### 4.2 推荐启动方式（目标机）

使用固定工作目录，便于集中存放日志：

```bash
cd /var/log/transferFile   # 或 /data/transfer/log
mkdir -p log
/path/to/transferFile -c /etc/transferFile/transferFile.gateway.target.json
# 实际日志: /var/log/transferFile/log/2026-06-01.log
```

当前代码中 `init("log")` 写死为相对路径 `log`；若需绝对路径需改 `main.cpp` / `platform_sim_main.cpp` 中的 `init` 参数（V0.0.1 未做配置项）。

### 4.3 版本控制

`log/` 已加入仓库 `.gitignore`，不会提交运行产物。

## 5. 生命周期与 API

### 5.1 源码位置

| 文件 | 说明 |
|------|------|
| `include/transfer/runtime_log.hpp` | 对外 API |
| `src/app/runtime_log.cpp` | 实现 |

### 5.2 调用约定

```cpp
// 程序入口 early（main 开头）
transfer::log::init("log");

// 程序退出前（main 使用 RAII LogGuard 自动 shutdown）
transfer::log::shutdown();
```

### 5.3 写日志（业务代码）

```cpp
#include "transfer/runtime_log.hpp"

transfer::log::gatewayInfo("服务就绪：等待平台通过 MQTT 下发召唤");
transfer::log::gatewayWarn("传输超时中止 CmdId=" + std::to_string(cmdId));
transfer::log::gatewayError("召唤解析失败: " + errDetail);

transfer::log::platformInfo("<<< 已发布召唤 -> " + topic);
```

`gateway*` 用于网关侧，`platformInfo` 用于 `platform_sim`。文件与行号由编译器内置函数自动填充，**无需**手动传入。

底层接口（一般不需直接调用）：

```cpp
void writeGateway(Level level, const char* file, int line, const std::string& msg);
void writePlatform(Level level, const char* file, int line, const std::string& msg);
```

## 6. 典型消息对照（网关 transferFile）

按一次成功传输的时间顺序：

| 阶段 | 级别 | 典型消息（摘要） | 源模块 |
|------|------|------------------|--------|
| 启动 MQTT | INFO | `正在连接 MQTT Broker ...` | mosquitto_mqtt_adapter |
| 已连接 | INFO | `MQTT 已连接 Broker，已订阅: ...` | mosquitto_mqtt_adapter |
| 就绪 | INFO | `服务就绪：等待平台通过 MQTT 下发召唤` | main |
| 收到召唤 | INFO | `<<< 收到召唤报文，长度 N 字节` | transfer_orchestrator |
| 解析召唤 | INFO | `解析召唤 CmdId=... 文件=... StartByte=...` | transfer_orchestrator |
| 简报失败 | INFO | `>>> 已发布简报失败 CmdId=... ErrorCode=...` | transfer_orchestrator |
| 简报成功 | INFO | `>>> 已发布简报成功 CmdId=... FileSize=...` | transfer_orchestrator |
| 发送内容 | INFO | `开始发送文件内容...` | transfer_orchestrator |
| 内容段 | INFO | `已发送内容段 CmdId=... FileSegNo=...` | transfer_orchestrator |
| 完成 | INFO | `传输完成 CmdId=... 总大小=...` | transfer_orchestrator |
| 解析失败 | ERROR | `召唤解析失败: ...` | transfer_orchestrator |
| 读文件失败 | ERROR | `读取文件失败，传输中止 CmdId=...` | transfer_orchestrator |
| 超时 | WARN | `传输超时中止 CmdId=...` | transfer_orchestrator |
| 订阅失败 | ERROR | `订阅召唤 Topic 失败: ...` | mosquitto_mqtt_adapter |

模拟总线模式（`useSimulatedBus=true`）另有 `模拟 MQTT 总线已启动`、`模拟总线收到召唤` 等。

## 7. 典型消息对照（平台 platform_sim）

| 阶段 | 典型消息（摘要） |
|------|------------------|
| 发布召唤 | `<<< 已发布召唤 -> <topic>` |
| 目标机联调提示 | `【目标机网关】召唤路径(须在目标机已存在): ...` |
| 监听中 | `监听网关应答中...` |
| 简报摘要 | `简报摘要 成功/失败 ...`（动态拼接） |
| 联调结果 | `联调成功：平台发召唤，目标机网关已回传文件` |

## 8. 联调与运维

### 8.1 实时查看

```bash
# 在 transferFile 工作目录下
tail -f log/$(date +%Y-%m-%d).log

# 或过滤关键字
tail -f log/$(date +%Y-%m-%d).log | grep -E '召唤|简报|传输完成|ERROR|WARN'
```

### 8.2 与 MQTT 联调文档的关系

拓扑、Broker、配置文件见 [10-MQTT本机联调.md](10-MQTT本机联调.md)。本节仅说明**如何从日志判断进度**。

| 期望现象 | 日志线索 |
|----------|----------|
| 网关已连 Broker | `MQTT 已连接 Broker` |
| 平台已发召唤 | 平台：`已发布召唤`；网关：`收到召唤报文` |
| 文件可读 | 网关：`简报成功`（非 `FILE_NOT_FOUND` 等） |
| 传输结束 | 网关：`传输完成`；平台：`联调成功` |
| 超时 | 网关：`传输超时中止` |

### 8.3 常见问题

| 现象 | 可能原因 | 处理 |
|------|----------|------|
| 无 `log/` 目录 | 未调用 `init` 或进程无写权限 | 检查工作目录权限；确认使用带 RuntimeLog 的二进制 |
| `log/` 为空但有控制台输出 | `init` 失败前即退出，或磁盘满 | 检查 `df`、目录权限 |
| 路径显示为 `main.cpp:182` 而非 `src/...` | 编译未保留路径信息 | 正常；Release 下 `-fmacro-prefix-map` 等可能影响显示 |
| 交叉编译后无日志文件 | 目标机只读文件系统 | 将工作目录设到可写分区（如 `/tmp`、`/data`） |
| 重复日期文件过大 | 长时间运行 | V0.0.1 无自动清理；运维定期归档或 `logrotate` |

## 9. 安全与隐私

- **不记录** Content 帧 Base64 载荷、完整召唤/简报 JSON（除 orchestrator 中的路径、CmdId 等元数据）。
- **会记录** `FullPathFileName` 等路径字符串，部署时注意日志目录访问权限。
- 生产环境若路径敏感，后续版本可增加脱敏或关闭文件落盘开关。

# 01 — 需求说明

## 1. 文档目的

将项目 V0.0.1 的功能需求、非功能需求及验收标准形式化，作为设计与后续 TDD 实现的依据。本文档对应仓库 `.cursorrules` 中的业务描述，并补充可测试的验收条目。

## 2. 设计假设（审阅时请确认）

1. **实现角色**：仅实现**网关侧**；平台侧由既有系统承担，不在本仓库实现。
2. **传输模型**：单网关单进程；V0.0.1 默认**同一时刻仅处理一个活跃传输会话**（新召唤在忙时拒绝或返回「忙」— 具体错误码待协议，见 [04-通信协议.md](04-通信协议.md)）。
3. **断点续传**：由**平台再次下发召唤**，携带相同 **`CmdId`**（建议）、`FullPathFileName` 及 **`StartByte`（1-based）**；网关校验后从对应偏移继续，**不主动**发起续传。
4. **文件定位**：召唤 JSON 使用 **`FullPathFileName`** 全路径；网关须校验路径落在配置的允许根目录内，禁止越权访问（见 [04-通信协议.md](04-通信协议.md)）。
5. **有效交互（超时相关）**：协议**无平台 ACK**；超时计时仅由**新的合法召唤**重置，否则 180s 后中止（见 [03-业务流程与状态机.md](03-业务流程与状态机.md)、[04-通信协议.md](04-通信协议.md) 第 8 节）。

## 3. 功能需求

### R1 — 平台通过 MQTT 下发召唤文件指令

| 项 | 内容 |
|----|------|
| 描述 | 网关订阅约定 Topic，接收并解析平台下发的召唤文件指令，进入传输流程。 |
| 优先级 | 必须 |
| 验收标准（文档级） | ① 能建立 MQTT 连接并订阅召唤 Topic；② 收到合法召唤后触发会话创建或续传校验；③ 非法帧（校验失败、字段越界）不进入发送内容阶段，按协议返回简报失败或丢弃并记录日志（具体策略在协议定稿时二选一写死）。 |

### R2 — 先应答文件简报，再应答文件内容

| 项 | 内容 |
|----|------|
| 描述 | 在同一传输会话内，网关必须先向平台发送**文件简报应答**，之后才能发送**文件内容**数据。 |
| 优先级 | 必须 |
| 验收标准 | ① 任意成功进入内容发送的会话，其时间序上存在且仅存在一条「简报成功」之后的 Content 序列；② 单元/集成测试可断言：未发送简报前 `encodeContent` / 发布内容 Topic 的调用次数为 0。 |

### R3 — 无法传输时仅在简报中失败，不发送内容

| 项 | 内容 |
|----|------|
| 描述 | 若网关因任何原因当前无法传输文件（文件不存在、无权限、正在处理其他会话、磁盘错误等），仅在简报应答中返回失败，**不得**再发送任何文件内容帧。 |
| 优先级 | 必须 |
| 验收标准 | ① 所有「不可传」分支仅产生简报失败；② 该会话下内容 Topic 发布次数为 0；③ 状态机进入终止态（见 [03-业务流程与状态机.md](03-业务流程与状态机.md)）。 |

### R4 — 超时未响应则中止传输

| 项 | 内容 |
|----|------|
| 描述 | 在传输会话进行中，若超过规定时间未发生「有效交互」，则中止当前传输。 |
| 优先级 | 必须 |
| 默认超时 | **180 秒（3 分钟）**，可通过配置项 `timeoutSec` 调整。 |
| 计时起点 | 会话创建时启动；每次**有效交互**发生时重置（有效交互定义见 [04-通信协议.md](04-通信协议.md)，暂列 ACK / 新合法召唤）。 |
| 验收标准 | ① 超时后会话进入 Aborted，停止后续内容发送；② 释放文件句柄与会话资源；③ 测试可用 Mock 时钟在小于真实 180s 的时间内触发超时逻辑。 |

### R5 — 支持断点续传

| 项 | 内容 |
|----|------|
| 描述 | 传输中断后，平台可再次下发召唤，携带**会话标识**（和/或文件标识）及**偏移**；网关校验通过后从该偏移继续读取并发送内容，不重复发送偏移之前的字节（推荐行为，若协议另有规定以协议为准）。 |
| 优先级 | 必须 |
| 验收标准 | ① 偏移合法且文件未变更时，首帧内容对应偏移位置；② 已发送字节数 + 本次发送 = 直至文件尾；③ 非法偏移或会话不匹配时简报失败且不发送内容。 |

## 5. 非功能需求

### 5.1 开发语言与规范

- 使用 **C++** 实现；标准版本在 [07-构建与交叉编译.md](07-构建与交叉编译.md) 中定为 **C++17**（兼顾 GCC 8.3 与可读性）。
- **源代码注释使用中文**，说明非显而易见的业务规则与协议约束。

### 5.2 构建与部署

- 使用 **CMake** 组织工程。
- 支持 **交叉编译**，工具链包路径：`/home/huangzd/openwrt-gcc-8.3.0.tar.gz`（解压与配置步骤见 07 章）。

### 5.3 工程质量

- 以 **TDD** 为主：先写失败测试 → 实现 → 测试通过 → 重构。
- 项目目录遵循 C++ 常见实践（见 07 章规划）。

### 5.4 文档

- 设计类文档全部位于 `document/` 目录。

### 5.5 可维护性

- 协议编解码与 MQTT、文件 IO 解耦，便于 Mock 与单测（见 [05-模块与接口设计.md](05-模块与接口设计.md)）。

## 6. V0.0.1 明确不包含的范围

| 排除项 | 说明 |
|--------|------|
| 平台侧实现 | 不实现召唤下发、内容接收存储的完整平台服务 |
| 文件加密/压缩 | 载荷为原始字节，除非协议层另有定义 |
| 多文件并发传输 | 默认单会话串行；并发作为后续版本评估 |
| 传输过程中的文件写入 | 网关侧为**读文件上传**场景 |
| 具体协议字节布局 | 由业务方后续提供，不在本阶段臆造 |

## 7. 配置项汇总（已实现）

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `timeoutSec` | uint32 | 180 | 会话超时秒数 |
| `chunkSize` | uint32 | 4096 | 单段内容最大字节 |
| `allowedPathRoots` | string[] | `/tmp/` 等 | 见 `config/*.json` |
| `mqtt.*` | — | 见 09 章 | Broker、Topic、`useSimulatedBus` |

运行日志：控制台 + `log/YYYY-MM-DD.log`，详见 [11-运行日志.md](11-运行日志.md)。

# 09 — 配置文件说明

## 1. 配置文件一览

| 文件 | 用途 |
|------|------|
| [config/transferFile.json](../config/transferFile.json) | 默认（程序未指定 `-c` 时尝试加载） |
| [config/transferFile.json.example](../config/transferFile.json.example) | 字段说明示例 |
| [config/transferFile.mqtt-debug.json](../config/transferFile.mqtt-debug.json) | **网关 + 平台都在本机**，Broker `127.0.0.1` |
| [config/transferFile.platform.json](../config/transferFile.platform.json) | 开发机 **platform_sim** |
| [config/transferFile.gateway.target.json](../config/transferFile.gateway.target.json) | **目标机 transferFile**（`brokerHost` 为开发机 IP） |

启动：

```bash
./transferFile -c config/transferFile.gateway.target.json
./platform_sim -c config/transferFile.platform.json --gateway-file /tmp/platform_test_file.bin
```

路径不存在时使用内置默认值，并在启动横幅中提示。

## 2. JSON 结构

```json
{
  "transfer": { ... },
  "mqtt": { ... }
}
```

### 2.1 transfer（传输参数）

| 字段 | 类型 | 说明 | 默认 |
|------|------|------|------|
| timeoutSec | string | 会话超时（秒） | 180 |
| chunkSize | string | 单段内容最大字节数 | 4096 |
| allowedPathRoots | string 数组 | 允许读取的文件路径前缀 | `["/tmp/"]` |

目标机联调常见配置：`["/tmp/", "/data/transfer/"]`。

### 2.2 mqtt（MQTT 参数）

| 字段 | 类型 | 说明 | 默认 |
|------|------|------|------|
| brokerHost | string | Broker 地址 | 127.0.0.1 |
| brokerPort | string | Broker 端口 | 1883 |
| gatewayId | string | 网关 ID，用于生成 Topic | gw001 |
| clientId | string | MQTT 客户端 ID；空则自动生成 | gateway-transfer-sim-{gatewayId} |
| topicSummon / topicBrief / topicContent | string | 空则按 gatewayId 自动生成 | 见 04 章 |
| useSimulatedBus | string | `true`=内存模拟，`false`=libmosquitto | 见各配置文件 |
| username / password | string | 鉴权（可选） | 空 |
| qos | string | 0/1/2 | 1 |
| keepAliveSec | string | Keep Alive | 60 |

**重要**：`platform_sim` 与 `transferFile` 的 **`gatewayId` 必须一致**；`brokerHost` 各填**本机访问 Broker 的地址**（目标机通常填开发机局域网 IP）。

### 2.3 模式切换

| useSimulatedBus | 行为 |
|-----------------|------|
| `true` | `SimulatedMqttBus`，无需 Broker |
| `false` | `MosquittoMqttAdapter` 连接真实 Broker |

## 3. 修改示例

**目标机连开发机 Broker：**

```json
"mqtt": {
  "brokerHost": "192.168.1.100",
  "brokerPort": "1883",
  "gatewayId": "gw001",
  "useSimulatedBus": "false"
}
```

**仅允许指定目录：**

```json
"transfer": {
  "allowedPathRoots": ["/data/transfer/"]
}
```

## 4. 代码入口

- 加载：`loadAppConfig()` — `include/transfer/config_loader.hpp`
- 结构：`AppConfig` — `include/transfer/app_config.hpp`

## 5. 与运行日志

日志目录与配置文件无关，固定为进程工作目录下 `log/`（详见 [11-运行日志.md](11-运行日志.md)）。建议在目标机用 systemd 或启动脚本 `cd` 到固定目录，便于集中存放日志。

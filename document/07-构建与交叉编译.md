## 3. C++ 标准与编译选项

| 项 | 选择 |
|----|------|
| 标准 | **C++17**（`optional`、无 `std::filesystem` 于嵌入式路径） |
| 警告 | `-Wall -Wextra` |
| 编译数据库 | `CMAKE_EXPORT_COMPILE_COMMANDS=ON` |

**交叉编译注意**：`runtime_log` 使用 POSIX `mkdir` 创建日志目录，避免 GCC 8.3 + musl 下 `std::filesystem` 需链接 `libstdc++fs` 的问题。

## 4. 本地开发构建

```bash
sudo apt install mosquitto mosquitto-clients libmosquitto-dev   # 可选但推荐
./scripts/build-native.sh
```

产物：

| 文件 | 说明 |
|------|------|
| `build/transferFile` | 网关 |
| `build/platform_sim` | 平台模拟（仅主机） |
| `build/transfer_tests` | 单元测试 |

手动构建：

```bash
mkdir -p build && cd build
cmake .. -DCMAKE_BUILD_TYPE=Debug -DBUILD_TESTING=ON -DTRANSFER_WITH_MOSQUITTO=ON
cmake --build . && ctest --output-on-failure
```

## 5. 交叉编译（OpenWrt aarch64）

### 5.1 工具链

- 压缩包：`/home/huangzd/openwrt-gcc-8.3.0.tar.gz`
- 解压路径：`~/toolchains/openwrt-gcc-8.3.0`
- 编译器：`aarch64-openwrt-linux-musl-g++`

### 5.2 一键构建

```bash
./scripts/build-openwrt.sh
```

流程：先 `build-mosquitto-cross.sh` → `third_party/mosquitto-aarch64/` → CMake `build-openwrt/` → `transferFile`。

产物：`build-openwrt/transferFile`（ELF aarch64）。

部署到目标机：`transferFile`、配置文件、若动态链接则需 `libmosquitto.so*`（见 `third_party/mosquitto-aarch64/lib/`）。

### 5.3 编译警告

`STAGING_DIR not defined` 为 OpenWrt 工具链常见提示，**可忽略**，不影响链接。

## 6. CMake 目标

| 目标 | 类型 | 说明 |
|------|------|------|
| `transfer_core` | STATIC | 核心业务 + MQTT 适配 |
| `transferFile` | 可执行 | 网关入口 |
| `platform_sim` | 可执行 | 仅 `NOT CMAKE_CROSSCOMPILING` 时构建 |
| `transfer_tests` | 可执行 | 仅 `BUILD_TESTING=ON` 且非交叉编译 |

## 7. 依赖

| 依赖 | 用途 |
|------|------|
| CMake ≥ 3.14 | 构建 |
| libmosquitto | 真实 MQTT（`TRANSFER_WITH_MOSQUITTO`） |
| pthread | mosquitto |
| GoogleTest | 可选；无则 `minimal_test` |

## 8. IDE / 语言服务配置

消除「找不到 `transfer/xxx.hpp`」误报：

1. 先执行 `./scripts/build-native.sh` 生成 `build/compile_commands.json`
2. 仓库已含：
   - `.vscode/c_cpp_properties.json`、`settings.json`
   - `.clangd`（`CompilationDatabase: build`）
   - 根目录 `compile_commands.json` → `build/compile_commands.json`
3. Cursor/VS Code：**Reload Window**

## 11. 已知风险

| 风险 | 缓解 |
|------|------|
| 工具链路径不同 | 修改 `build-openwrt.sh` 中 `TOOLCHAIN_ARCHIVE` |
| 目标板无 libmosquitto.so | 使用脚本编出的静态/捆绑库，或拷贝 `third_party/mosquitto-aarch64/lib/` |
| Broker 仅监听 127.0.0.1 | 目标机 `brokerHost` 填开发机局域网 IP；或配置 mosquitto 监听 `0.0.0.0` |
